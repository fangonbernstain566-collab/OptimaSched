Date: 2026-07-06
Feature: Curriculum-Based Subject/Class-Code Filtering + Timetable Allocation Progress Widget
============================================================

## Feature Summary
Schedules can now only be built from subjects that actually belong to the selected section's curriculum, instead of picking from every subject offering in the system. This is powered by a new `CurriculumSubject` model mapping program + year level to subjects, and a new `classCode` field on `SubjectOffering` (e.g. "IT101-A") that identifies the specific class section a schedule slot belongs to. The dashboard's placeholder "Timetable Allocation Status" panel was replaced with a live widget showing real allocation progress for the active school year/semester.

## Changes Made
### Frontend
- `Schedules.jsx`: Section select now cascades — choosing a Section fetches that section's curriculum subjects via `/api/subject-offerings/by-section/:sectionId`; Subject and Class Code are now two dependent dropdowns instead of one flat Subject Offering picker; added validation requiring Section, Subject, and Class Code before submit; table now shows a Class Code column.
- `ScheduleManager.jsx`: added a Subject Offering dropdown plus a read-only Class Code display field; table shows Class Code column.
- `SchedulePlotter.jsx`: tooltip and placement-confirmation dialog now display the class code alongside the subject.
- `ClassDetailsModal.jsx`: shows the subject's code under the subject name.
- `DashboardView.jsx`: swapped the static placeholder panel for the new `TimetableAllocationProgress` component.
- Added `TimetableAllocationProgress.jsx`: self-contained widget with loading skeleton, error/retry state, a progress bar, and Allocated/Remaining/Completed-Blocks stat tiles, fed by `/api/dashboard/timetable-progress`.

### Backend
- Added `subjectOffering.routes.js` — `GET /api/subject-offerings/by-section/:sectionId` resolves a section's program + year level, looks up its `CurriculumSubject` mappings, and returns subjects grouped with their nested class codes.
- `dashboard.routes.js` — added `GET /api/dashboard/timetable-progress`, computing allocation percentage from sections with at least one active schedule in the current school year + semester.
- `schedule.controller.js` — `getScheduleOptions` now also returns `subjectOfferings` (with subject included) in dropdown data.
- `app.js` — mounted the new subject-offering routes at `/api/subject-offerings`.

### Database
- `Section`: added `program` (String) and `yearLevel` (Int), backfilled from the existing `"<PROGRAM>-<YEAR_LETTER><BLOCK>"` naming convention.
- `SubjectOffering`: added a required, unique `classCode` field; existing rows backfilled as `"<SubjectCode>-<Letter>"`.
- Added `CurriculumSubject` model (program, yearLevel, subjectId) with a unique constraint on the triple, mapping which subjects belong to which program/year level.
- Migration includes a dev-only bootstrap that maps every existing Subject to all 4 BSIT year levels so old seed data keeps working until real mappings are entered per subject.
- `seed.js` rewritten to seed a full BSIT Year 1-4 curriculum (subjects, offerings with real class codes, and curriculum mappings) instead of a single hardcoded subject, and to set `program`/`yearLevel` on seeded sections.

### API
- New: `GET /api/subject-offerings/by-section/:sectionId`
- New: `GET /api/dashboard/timetable-progress`
- Changed: `GET /api/schedules/options` response now includes `subjectOfferings`

### Validation
- `Schedules.jsx` form now blocks submission unless Section, Subject, and Class Code are all selected.

### Bug Fixes
- None.

## Technical Flow
1. User opens the Schedules form and picks a Section.
2. Frontend calls `GET /api/subject-offerings/by-section/:sectionId`; backend resolves the section's `program`/`yearLevel`, queries `CurriculumSubject` for matching subjects, then fetches their `SubjectOffering` rows and groups them by subject with nested class codes.
3. User picks a Subject (from the filtered list), then a Class Code (from that subject's offerings) — selecting a Class Code sets `formData.subjectOfferingId`, the value actually submitted with the schedule.
4. On the dashboard, `TimetableAllocationProgress` calls `GET /api/dashboard/timetable-progress` on mount, which counts total sections vs. sections with at least one non-deleted schedule in the current school year + semester, and renders a progress bar and stat tiles.
